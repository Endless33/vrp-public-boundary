# Protected Runtime

## Purpose

The Protected Runtime is the proprietary execution environment of the VRP ecosystem.

It is responsible for producing deterministic runtime behavior while preserving implementation confidentiality.

This repository intentionally excludes the Protected Runtime.

---

## Why The Runtime Is Protected

The runtime represents years of engineering, validation and implementation work.

Publishing the implementation is not required for independent engineering evaluation.

Instead, VRP separates observable engineering behavior from proprietary runtime execution.

---

## Public Engineering

External engineers may evaluate:

- engineering documentation
- public contracts
- SDK interfaces
- validation specifications
- evidence formats
- integration workflows
- Pilot documentation

These materials are sufficient for technical evaluation.

---

## Protected Engineering

The following components remain proprietary.

- Runtime Core
- Session Engine
- Authority Engine
- Recovery Engine
- Migration Engine
- Replay Protection
- Runtime Scheduler
- Internal Mathematics
- Runtime Decision Logic
- Commercial Runtime Components

---

## Engineering Boundary

The public engineering boundary explains how to interact with VRP.

The Protected Runtime determines how VRP internally operates.

The separation is intentional.

---

## Commercial Access

Authorized Pilot engagements may include additional runtime capabilities under separate commercial agreements.

Protected implementation is never distributed through this repository.
