# Open Engineering Model

## Overview

VRP follows an open engineering model rather than an open runtime model.

The engineering ecosystem is intentionally separated from the proprietary runtime implementation.

This separation allows independent technical evaluation while preserving implementation integrity.

---

## Public Engineering

The following components may be published.

- documentation
- SDK interfaces
- public contracts
- schemas
- validation specifications
- evidence specifications
- API documentation
- deployment guidance
- Pilot documentation
- engineering examples

---

## Protected Engineering

The following components remain proprietary.

- Protected Runtime
- runtime scheduler
- authority engine
- recovery engine
- migration engine
- replay engine
- runtime mathematics
- decision engine
- optimization logic
- commercial runtime

---

## Engineering Objective

Enable engineers to integrate with VRP.

Do not require engineers to possess proprietary implementation.

---

## Engineering Philosophy

Open interfaces.

Protected implementation.

Independent validation.

Professional collaboration.
