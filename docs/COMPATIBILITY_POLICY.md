# Compatibility Policy

## Purpose

Compatibility is an essential engineering requirement for every public component of the VRP ecosystem.

---

## Objectives

Public engineering artifacts should remain predictable across future releases.

Backward compatibility is preferred whenever practical.

---

## Covered Components

This policy applies to:

- SDK interfaces
- API specifications
- public contracts
- schemas
- validation specifications
- evidence formats

---

## Versioning

Breaking changes require a new major version.

Minor additions should remain compatible with previous public releases.

---

## Runtime Independence

Compatibility of the public engineering boundary does not imply identical internal runtime implementation.

The Protected Runtime may evolve independently.

---

## Engineering Principle

Stable contracts reduce integration complexity.

Implementation flexibility preserves long-term innovation.
