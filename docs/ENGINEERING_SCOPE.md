# Engineering Scope

## Purpose

This document defines the engineering scope of the VRP Public Engineering Boundary.

Its objective is to clearly distinguish public engineering artifacts from proprietary runtime technology.

---

## Public Engineering Scope

The public engineering boundary may include:

- engineering documentation
- integration guides
- SDK interfaces
- API specifications
- public contracts
- schemas
- validation specifications
- evidence specifications
- Pilot documentation
- deployment concepts
- compatibility guidance
- engineering terminology

---

## Outside The Scope

This repository does not contain:

- Protected Runtime
- runtime implementation
- proprietary algorithms
- runtime mathematics
- authority engine
- recovery engine
- session engine
- migration engine
- replay engine
- internal schedulers
- customer-specific runtime packages

---

## Engineering Objective

The repository exists to support:

- evaluation
- integration
- interoperability
- validation
- Pilot preparation

without exposing proprietary implementation.

---

## Long-Term Maintenance

The engineering scope will continue expanding through additional public documentation and interfaces.

The Protected Runtime will continue evolving independently.
