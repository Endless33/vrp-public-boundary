# Contributing

## Purpose

Thank you for your interest in the VRP Public Engineering Boundary.

This repository documents the public engineering surface of the VRP ecosystem.

Contributions are welcome when they improve public engineering artifacts without affecting the Protected Runtime.

---

## Suitable Contributions

Examples include:

- documentation improvements
- typo corrections
- clarification of engineering concepts
- OpenAPI improvements
- JSON Schema improvements
- Protocol Buffers improvements
- SDK example improvements
- integration examples
- compatibility documentation
- engineering diagrams

---

## Contributions That Will Not Be Accepted

This repository does not accept contributions involving:

- Protected Runtime implementation
- proprietary algorithms
- runtime mathematics
- authority engine
- recovery engine
- migration engine
- replay engine
- commercial runtime packages
- customer-specific deployment material

---

## Pull Requests

Before submitting a Pull Request:

- review existing documentation;
- keep changes focused;
- maintain implementation independence;
- preserve compatibility whenever practical;
- avoid unrelated formatting changes.

Large architectural proposals should first be discussed through GitHub Issues.

---

## Documentation Standard

Public documentation should be:

- technically accurate;
- implementation independent;
- reproducible;
- concise;
- professionally written.

---

## Engineering Principle

The objective of this repository is to improve interoperability, documentation quality, and engineering transparency.

The objective is not to disclose proprietary runtime implementation.

---

## License

By contributing to this repository, you agree that accepted contributions become part of the public engineering boundary under the repository license.
